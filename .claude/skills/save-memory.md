# /save-memory - セッション記録の保存

現在のセッションの要約を `.claude/memory/sessions/` に保存する。

## 手順

1. 今日の日付でファイルパスを決定: `.claude/memory/sessions/YYYY-MM-DD.md`

2. 現在のセッション内容を以下の構造で要約:

```markdown
## セッション YYYY-MM-DD HH:MM

### 実施内容
- （具体的に何を実装・修正・議論したか箇条書き）

### 変更ファイル
- `path/to/file.py` — 変更内容の1行要約

### 設計判断・決定事項
- （このセッションで決まったこと、選んだ方針）

### 未着手・保留
- （議論したが未実装のもの、次回やること）

### メモ
- （ユーザーの好み、注意点、再現に必要な文脈）
```

3. 同日のファイルが既にある場合は**末尾に追記**（上書きしない）

4. ファイルが存在しない場合は新規作成。先頭に `# YYYY-MM-DD セッションログ` を付ける

5. 保存後、内容をユーザーに表示して確認

## 要約のルール

- **簡潔に**: 1セッション分は20-40行程度
- **具体的に**: 「UIを改善」ではなく「読込ビューにフィルタUI（銘柄・TF・データソース）を追加」
- **再現可能に**: 別PCの新セッションで読んで文脈が分かるレベル
- **コードは書かない**: ファイル名と変更概要のみ。コード断片は不要
- **ユーザーの意図を記録**: なぜその変更をしたか（ユーザーの発言ベース）

## 注意

- 作業ディレクトリの `.claude/memory/sessions/` に保存
- 出力は日本語
- 保存後のコミット・プッシュはユーザーに委ねる（自動では行わない）
