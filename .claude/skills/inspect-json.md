# /inspect-json - 結果JSONの詳細分析

指定した最適化結果JSONファイルを読み込んで詳細分析する。

## 手順

1. 引数で指定されたJSONファイルを `results/` から読み込む
   - ファイル名の一部でもマッチすればOK（例: `/inspect-json BTC` → BTCUSDT含むファイルを候補表示）
   - 複数マッチした場合は候補一覧を表示してユーザーに選択させる
2. 基本情報を表示:
   - 銘柄、実行TF、HTF、データソース、期間、保存日時
   - 結果件数、テンプレート数、レジーム数
3. レジーム別のベスト戦略を表示:

```
## レジーム別ベスト

| レジーム   | テンプレート       | スコア | PF   | 勝率  | DD%  | トレード数 |
|-----------|-------------------|--------|------|-------|------|-----------|
| uptrend   | rsi_pullback_long | 0.672  | 2.15 | 58.3% | 8.2  | 24        |
| downtrend | bb_short          | 0.583  | 1.82 | 55.1% | 12.4 | 31        |
| range     | mean_revert_long  | 0.445  | 1.45 | 52.0% | 15.1 | 18        |
```

4. 品質チェック:
   - トレード数10未満 → ⚠️ 統計的に弱い
   - DD > 20% → ⚠️ 高ドローダウン
   - PF < 1.0 → ⚠️ マイナス期待値
   - 勝率 < 40% → ⚠️ 低勝率
5. スコア分布の概要（上位5件、下位5件）
6. 採用推奨パラメータの提示
